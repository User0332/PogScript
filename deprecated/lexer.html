Section 1: Lexical Analysis

1a. Basic Keywords

let, def {ident}

Creates a dynamic variable/function which can be referenced from ident. This does not affect runtime, but it may cause undefined behavior as it is represented as an integer (or consecutive integers in memory) and has no defining label to represent the datatype or return type.

from libc import puts

int main(int argc, string argv[])
{
        let mystr = "Hello World!"
        puts(mystr)

        return 0
}

Output:

Hello World!

class {ident}

Creates a class referenced by ident.

struct {ident}

Creates a struct referenced by ident.

Modifies the accessibility and view permissions of attribute. Theoretically, one could bypass the access modifiers, but these are put in place to ensure proper use of classes. All attributes are private by default. Below is a table of access modifiers and their level of privacy.

from {module} import {*}|{attribute}

Imports attribute attribute from the module module. If a (*) is given, all the attributes of the module are imported. (*) and attribute are mutually exclusive. This is the preferred keyword to use over the include directive.

return {value}

Returns value, which may be an expression, to the caller of the function. Can only be used inside of a function. If a function’s return type is void and an expression is provided, a warning will be thrown and the expression will be discarded. To return from a void returning function, omit the expression after the return keyword.

1b. Special Keywords

{p} = memresb {n}

Reserves n bytes of memory and puts the starting address in pointer p. Can be used to statically allocate memory before runtime for use during the program and for use during functions. This is not the malloc() function found in <stdlib.h>. Equivalent of p resb n in the assembly output file. Ex:

from libc import scanf, printf

int main(int argc, string argv[])
{
        strptr = resb 20
        scanf("%20s", strptr)
        printf("Hello, %s!\n", strptr)

        return 0

}

Compiler Output:

extern _scanf
extern _printf

section .text:
global _main
_main:
        push strptr
        push .LC.S0
        call scanf
        push .strptr
        push .LC.S1
        call printf

        mov eax, 0
        ret

.LC.S0: db "%20s", 0
.LC.S1: db "Hello, %s!\n", 0
.strptr: resb 20

__asm {code}

Inline assembly. This provides full access, and is the equivalent of editing the output file. This should only be used when the user knows what they are doing, as stack and register operations may result in undefined behavior.         The item code should be a string literal (not a pointer to a string) and should contain one line of assembly, e.g. one instruction. Ex:

from libc import printf


int main(int argc, string argv[])
{
        printf("Hello World!\n")
        return 0
}

is the same as

__asm "extern _printf"

int main(int argc, string argv[])
{
        __asm ".LCS0 db \"Hello World!\", 0xA, 0"
        __asm "push .LCS0"
        __asm "call _printf"
        __asm "mov eax, 0"
        __asm "ret"
}

__sizeof {obj}

Returns the size, in bytes, of the object obj, in the form of an int. Ex:

from libc import printf

int main(int argc, string argv[])
{
        printf("%i", __sizeof int)
        

        return 0
}

Output:

4

__typeof {obj}

Returns the type of the object obj, in the form of a class, struct, or primitive type.

from libc import printf

int main(int argc, string argv[])
{
        int x = 7
        __typeof(x) y = 8
        printf("%i", y)
        

        return 0
}

Output:

8

memaddr {obj}

Gets the memory address of obj. Similar to the & operator in C, it can be used to assign a memory address to a pointer.

from libc import printf
int main(int argc, string argv[])
{
        int x = 7
        int* xptr = memaddr x

        printf("%p", xptr)

        return 0
}

Output:

7

1c. Lexer Directives

using {syntaxcustomization}

Tells the lexer to add syntaxcustomization to the list of syntax customizations. These include:

braces - Brace delimited scopes (as opposed to indents)

semicolons - Semicolon delimited expressions (as opposed to newlines)

include {filename}

Includes the contents of the file referenced by filename into your current file. This is usually used for small local libraries or a file of constants. The import keyword is typically preferred over the include directive.
